Motion-Primitives = Framer Motion patterns, dropped in.
It's a collection of pre-built React + Framer Motion components — text reveals, scroll-triggered transitions, hover micro-interactions. Copy-paste, no install ceremony. shadcn-compatible.
→ MIT · React + Tailwind

02 · meet
Eldora UI = pure CSS animations, zero JS.
Same energy, different stack: Eldora ships motion as CSS keyframes + Tailwind. No Framer dependency, no JS bundle hit. Use when you need animation but can't afford the runtime.
→ MIT · Tailwind only

03 · how
Both drop straight onto shadcn.
Neither replaces shadcn — both extend it. Your buttons, dialogs, dropdowns stay shadcn. The hero text, the card hover, the section reveals — those become Motion-Primitives or Eldora.

05 / 08 — 04 · pick

04 · pick
When to pick which.
Need rich gesture + spring physics → Motion-Primitives. Need lightweight CSS-only animations that work without React → Eldora. Need both → use both. They don't fight.

05 · install
Install in one command.
Motion-Primitives: `npx motion-primitives@latest add text-reveal`. Eldora: pure copy-paste from the site. Both work alongside your existing shadcn registry.
$ npx motion-primitives add text-reveal
