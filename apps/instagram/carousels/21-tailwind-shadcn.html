01 · tailwind
Tailwind v4 is native CSS now.
v4 dropped the JS config — everything is CSS variables, processed at build time. Smaller runtime, faster Vite/Next builds, easier to theme. If you're still on v3 you're already behind.
→ 5× faster builds · zero JS at runtime

02 · registry
shadcn registry = npm for components.
Publish your own component registry (or use a team's), then anyone runs `npx shadcn add @yourcompany/button` and gets your version of Button — code dropped into their repo, not a dep.
npx shadcn add @yourcompany/button

04 / 08 — 03 · why
post · 21ui · design systems
03 · why
Bring-your-own design system.
Old way: build a component library, publish to npm, version-lock all consumers. New way: publish a registry, consumers copy the code, own it, modify it. No version hell.
→ code over deps · always

05 / 08 — 04 · setup
post · 21ui · design systems
04 · setup
Set up your own in 10 minutes.
Create a registry.json with your component manifests, host it on Vercel (or any static), point shadcn CLI at it. Done. Your whole team can `shadcn add` from your URL.
